# Contributing

Thanks for your interest in contributing to Learning Organisation 13. This organisation is primarily a personal learning space focused on data engineering, but feedback and contributions are welcome.

 ## Ways to Contribute

You can contribute by:
- Suggesting project ideas or improvements
- Reporting bugs or inconsistencies
- Improving documentation or examples
- Enhancing templates or learning notes
If you’re unsure whether something fits, feel free to open a discussion first.

## Contribution Workflow

1. Fork the repository
2. Create a feature branch
   git checkout -b feature/your-change
3. Make your changes with clear, focused commits
4. Open a pull request explaining:
   - What you changed
   - Why it’s useful

## Guidelines

- This is a learning-first environment; clarity is more important than perfection
- Follow existing project structure and naming conventions
- Keep code readable and well-documented
- Prefer simple, explicit solutions

## Code of Conduct

Be respectful and constructive in all discussions.
Assume good intent and provide feedback that helps others learn.
